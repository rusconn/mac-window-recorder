# 色問題の調査まとめ

## 結論

AVAssetWriterに`sRGB + BT.709`を正しく設定すれば、色補正は不要。SCStreamの`colorSpaceName = CGColorSpace.sRGB`とAVAssetWriterの`AVVideoColorPropertiesKey = BT.709`の組み合わせで、mpv上では実ウィンドウとほぼ完全に一致する。

## 調査で判明したこと

### 色の問題の本質

1. **colorSpaceName未指定時**: Display P3相当のネイティブ色空間の値が渡される
2. **P3データにBT.709タグだけ付与**: 実際の中身(P3)とタグ(bt709)が食い違い、赤・緑方向に色がズレる

### テスト結果

| 設定 | 赤偏差 | 緑偏差 | 青偏差 | 結論 |
|------|--------|--------|--------|------|
| カラースペース指定なし + BT.709タグなし | R-22 | G+20 | - | ガマットシフト |
| P3指定 + BT.709タグなし | 上記と同一 | - | - | CVPixelBufferのアタッチメントは無視される |
| P3指定 + BT.709タグあり | 上記と同一 | - | - | - |
| sRGB指定 + BT.709タグなし | 上記と同一 | - | - | CVPixelBufferのアタッチメントは無視される |
| sRGB指定 + BT.709タグあり | ±1-2 | - | - | **採用（現行設定）** |

### AVAssetWriterの挙動

- `AVVideoColorPropertiesKey`を設定すると、BT.709 YCbCr変換が適用される
- 設定しない場合はraw passthroughで、ガマットシフトが発生する
- CVPixelBufferの`CVBufferSetAttachment`で設定した色空間はAVAssetWriterに無視される

### CVPixelBufferの色空間アタッチメントとBT.709タグの衝突

SCStreamが返すCVPixelBufferには色空間に関するアタッチメント（`kCVImageBufferColorSpaceKey`等）が付与されている。このアタッチメントがAVAssetWriterの`AVVideoColorPropertiesKey`（BT.709）と衝突し、映像が暗くなる問題が発見された。

#### 発生条件

| 処理 | 明るさ |
|------|--------|
| SCStreamのCVPixelBufferをそのまま`pixelBufferAdaptor.append()` | 暗くなる |
| SCStreamのCVPixelBufferをコピーして新規バッファ作成 → `pixelBufferAdaptor.append()` | 正常 |
| `CVBufferRemoveAttachment`でアタッチメント除去 → `pixelBufferAdaptor.append()` | 正常 |

#### 原因

- SCStreamのCVPixelBufferにはsRGB色空間のアタッチメントが付与されている
- `AVAssetWriterInputPixelBufferAdaptor.append()`で渡すと、エンコーダがこのアタッチメントを参照する
- エンコーダ設定のBT.709タグとアタッチメントのsRGBが衝突し、二重のガンマ補正が適用され暗くなる

#### 解決策

`CVBufferRemoveAttachment`でアタッチメントを直接除去し、コピー不要で正しく色再現する。

```swift
let pixelBuffer = CMSampleBufferGetImageBuffer(sampleBuffer)!
CVBufferRemoveAttachment(pixelBuffer, kCVImageBufferCGColorSpaceKey)
pixelBufferAdaptor.append(pixelBuffer, withPresentationTime: relativePTS)
```

- メモリコピー不要のため、メモリ帯域の圧迫がない
- 正常に色再現される
- `captureDynamicRange = .SDR`だけでは不十分（アタッチメントが残る）

##### 補足: 各アプローチの比較

| アプローチ | 色 | メモリコピー | 備考 |
|-----------|-----|------------|------|
| そのまま渡す | 暗くなる | なし | アタッチメントとBT.709衝突 |
| CVPixelBufferコピー | 正常 | あり | メモリ帯域圧迫 |
| `captureDynamicRange = .SDR` | 暗くなる | なし | アタッチメントが残る |
| `CVBufferRemoveAttachment` | 正常 | なし | **採用** |

#### 補足: CMSampleBuffer経由の場合

`CMSampleBufferCreateReadyWithImageBuffer`で新しいCMSampleBufferを作成しても、内部のCVPixelBufferは同じ参照のためアタッチメントが残り、暗くなる。根本的にCVPixelBuffer自体のアタッチメントを除去する必要がある。
